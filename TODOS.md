## TODO-1: Unmapped kit warning in TakeoffForm

**What:** When a TakeoffForm row selects an item whose kit has no match in the catalog, `expandKits` silently skips it — the quote total is wrong without any warning. Add an "unmapped" flag to the quote API response and show a warning icon on the affected row.

**Why:** A $0 materials total on a row the user just entered is a trust-destroying wrong number. An error is better than silence.

**Where to start:** `src/domain/calculators.ts:26` (`if (!kit) continue`) and `src/app/api/quote/route.ts`. Add a `warnings: string[]` field to `QuoteResult`, push `"unmapped: {id}"` for each skipped item, surface in `TakeoffForm`.

**Depends on:** TakeoffForm sprint (this sprint). Can be built as a follow-up in the same sprint or the next.

## TODO-2: Migrate catalog/kits data source from files to PocketBase

**What:** `POST /api/quote` and `GET /api/catalog` currently read from `data/sample-inputs/` (CSV + JSON files). PocketBase already has `products` and `item_kits` collections seeded. Migrate both routes to read from PocketBase. Add `takeoff_name` and `default_hours_per_unit` fields to `item_kits`.

**Why:** Files are a dev shortcut. PocketBase is the correct single source of truth. Without migrating, the seeded PocketBase data is unused by the app — the two sources can diverge silently.

**Where to start:** `src/app/api/quote/route.ts` (replace `loadCatalog`/`loadKits` with PocketBase SDK calls), `infra/pocketbase/pb_migrations/` (add migration for new fields), `scripts/seed-pocketbase.ts` (add default h/u values).

**Depends on:** This sprint's file-based catalog route being stable first.

## TODO-3: Replace North American sample catalog with Belgian/RGIE-compliant items

**What:** Current sample data uses North American conventions: `12/2 NMD` wire, `3-way switch`, `15A` outlets, `200A main panel`. Belgian standards use NYM/VVB wire types, 16A Type E outlets, and RGIE labor norms.

**Why:** The tool targets Belgian electricians. A Belgian estimator would not recognize the current catalog terminology and could not use the sample data for real work.

**Where to start:** `data/sample-inputs/sample-catalog.csv`, `data/sample-inputs/sample-kits.json`, `data/sample-inputs/sample-takeoff.json`. Replace with Belgian SKUs, Belgian supplier names (CEBO/Rexel Belgium), and RGIE h/u norms. Update all tests that assert specific dollar totals.

**Depends on:** TODO-2 (PocketBase migration) so the Belgian data lives in PocketBase, not files.

## TODO-D9: Keyboard row removal (Delete key)

**What:** Currently, × button is click-only (`tabIndex={-1}`). Add `onKeyDown` handler on each row `<tr>`: when focused + Delete/Backspace pressed, remove the row.

**Why:** Power users doing keyboard-only entry will want to remove a row without picking up the mouse. Keyboard shortcut is consistent with the keyboard-first premise.

**Pros:** 10-15 min to add. Satisfies strict keyboard-only users. Consistent with Tab-first flow.

**Cons:** Backspace on a focused row could accidentally delete if user expected to edit a field. Must gate on "no input is focused" within the row.

**Context:** Surfaced by /plan-design-review on 2026-06-24. Not required for 60-second success criteria.

**Depends on:** TakeoffForm sprint.
